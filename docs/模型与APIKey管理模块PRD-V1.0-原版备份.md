# 模型与 API Key 管理模块 PRD

> 所属产品：AI 漫剧生成器（PC Web）  
> 版本：V1.0  
> 模块定位：支持用户自主选择各环节 AI 模型，并自行配置对应 API Key 的能力中枢  
> 日期：2026-06-29

---

## 一、模块概述

### 1.1 背景
AI 漫剧生成器涉及多个 AI 能力环节：大语言模型、图像生成、视频生成、语音合成、音乐生成、音效生成等。不同用户对模型效果、成本、速度、稳定性的需求差异很大，单一模型无法满足所有场景。

### 1.2 目标
- 让用户可以自主选择每个创作环节使用的 AI 模型
- 让用户可以自行前往对应平台申请 API Key，并在产品中配置
- 让产品作为中立的创作编排平台，不绑定任何单一模型
- 保证 API Key 的安全存储与透明使用

### 1.3 设计原则
| 原则 | 说明 |
|------|------|
| 用户主权 | 用户选择模型、管理自己的 Key，平台不强制绑定 |
| 安全优先 | API Key 必须加密存储，调用过程不落日志 |
| 开放可扩展 | 新模型接入只需扩展配置，不改核心架构 |
| 透明计费 | 用户直接消耗各模型官方额度，平台展示估算费用 |
| 失败可恢复 | 支持重试、切换模型、切换 Key |

## 五、数据模型设计

### 5.1 AI 模型配置（AIModel）

```typescript
interface AIModel {
  id: string;                    // 模型唯一标识，如 "kling-pro"
  name: string;                  // 展示名称，如 "可灵 Kling Pro"
  provider: string;              // 提供商，如 "kling"
  capability: CapabilityType;    // 能力环节：llm | image | video | tts | music | sound
  icon: string;                  // 模型图标
  description: string;           // 模型简介
  docUrl: string;                // 官方文档链接
  pricingUrl: string;            // 计费说明链接
  apiBaseUrl?: string;           // API 基础地址（部分模型可自定义）
  parameters: ModelParameter[];  // 支持的参数列表
  status: 'active' | 'maintenance' | 'deprecated' | 'beta';
  supports: {
    referenceImage: boolean;     // 是否支持参考图
    firstFrame: boolean;         // 是否支持首帧控制
    lastFrame: boolean;          // 是否支持尾帧控制
    batchGeneration: boolean;    // 是否支持批量生成
  };
  billingUnit: 'token' | 'image' | 'second' | 'request';  // 计费单位
  billingRule?: BillingRule;     // 计费规则
}

type CapabilityType = 'llm' | 'image' | 'video' | 'tts' | 'music' | 'sound';

interface ModelParameter {
  key: string;
  name: string;
  type: 'select' | 'number' | 'string' | 'boolean';
  options?: { label: string; value: string | number }[];
  defaultValue: any;
  min?: number;
  max?: number;
  description?: string;
}

interface BillingRule {
  unitPrice: number;             // 单价
  currency: 'CNY' | 'USD';       // 币种
  unit: string;                  // 单位描述
}
```

## 七、关键交互逻辑

### 7.1 添加 API Key 流程

```
用户点击"添加模型/API Key"
    │
    ▼
选择能力环节（如：视频生成）
    │
    ▼
选择具体模型（如：可灵 Kling Pro）
    │
    ▼
展示该模型的官方文档和计费说明链接
    │
    ▼
用户点击"去官方获取 API Key"跳转新标签页
    │
    ▼
用户获取 Key 后返回本产品
    │
    ▼
粘贴 API Key
    │
    ▼
输入 Key 别名
    │
    ▼
设置默认参数
    │
    ▼
点击"验证 Key"
    │
    ▼
后端调用模型官方接口轻量验证（如生成一个测试任务或查询账户信息）
    │
    ├─ 验证通过 → Key 状态变为"有效" → 保存配置
    │
    └─ 验证失败 → 展示错误原因，提示检查 Key
```

### 7.2 任务执行时模型选择逻辑

```
生成任务触发
    │
    ▼
获取任务对应的能力环节（如 video）
    │
    ▼
优先读取项目级 ModelPreference 中该环节的 modelId + apiKeyId
    │
    ├─ 项目级已配置 → 使用项目级配置
    │
    └─ 未配置 → 读取全局默认 ModelPreference
        │
        ├─ 全局已配置 → 使用全局配置
        │
        └─ 未配置 → 提示用户先配置模型和 API Key
    │
    ▼
根据 modelId 读取模型配置，调用对应 Adapter
    │
    ▼
使用 apiKeyId 读取加密的 API Key 解密后请求模型 API
    │
    ▼
模型返回结果 → 保存任务结果 → 通知用户
```

### 7.3 失败重试逻辑

```
任务执行失败
    │
    ▼
判断错误类型：
    │
    ├─ 网络超时 / 模型服务端繁忙 → 标记为 retryable，自动重试（最多 3 次）
    │
    ├─ API Key 无效 / 额度不足 → 标记为不可重试，提示用户检查 Key
    │
    ├─ 参数不支持 / 内容审核不通过 → 标记为不可重试，提示用户修改参数
    │
    └─ 其他错误 → 记录日志，提示用户联系客服
    │
    ▼
若用户配置了备用 Key，自动切换备用 Key 重试
```

## 八、安全设计

### 8.1 API Key 安全

| 安全点 | 措施 |
|--------|------|
| 传输加密 | 全程 HTTPS，Key 在传输中加密 |
| 存储加密 | 数据库中 API Key 使用 AES-256-GCM 加密 |
| 密钥管理 | 加密密钥由 KMS/HSM 或环境变量管理，不硬编码 |
| 前端展示 | 永远只展示脱敏后的 Key，不允许完整复制 |
| 日志脱敏 | 日志中不记录完整 Key |
| 访问控制 | 用户只能查看/编辑自己的 Key |
| 审计日志 | 记录 Key 的增删改操作 |

### 8.2 模型调用安全

| 安全点 | 措施 |
|--------|------|
| 调用鉴权 | 服务端统一调用模型 API，不暴露 Key 给前端 |
| 限流熔断 | 对单个 Key、单个用户、单个模型设置限流 |
| 内容审核 | 对用户输入的提示词进行敏感内容检测 |
| 超时控制 | 设置合理的请求超时时间 |

## 九、错误处理

### 9.1 常见错误码

| 错误码 | 含义 | 用户提示 |
|--------|------|----------|
| KEY_INVALID | API Key 无效 | 您的 API Key 验证失败，请检查或重新配置 |
| KEY_EXPIRED | API Key 已过期 | 您的 API Key 已过期，请到官方平台续期 |
| QUOTA_EXCEEDED | 额度不足 | 您的 API 额度不足，请到官方平台充值 |
| MODEL_BUSY | 模型繁忙 | 模型当前繁忙，已自动重试，请稍候 |
| PARAM_INVALID | 参数不支持 | 当前模型不支持该参数，请调整 |
| CONTENT_REJECTED | 内容审核不通过 | 提示词包含不合规内容，请修改后重试 |
| TIMEOUT | 请求超时 | 请求超时，已自动重试 |
| MODEL_OFFLINE | 模型已下线 | 该模型已下线，请更换模型 |
| NO_DEFAULT_MODEL | 未配置默认模型 | 请先前往模型设置配置 API Key |
