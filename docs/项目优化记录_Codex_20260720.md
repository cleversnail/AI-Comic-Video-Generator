## 二、详细说明

### 2.1 QueueProcessor 接入 WebSocket 实时进度推送

**问题**：QueueProcessor 处理任务时只写数据库，前端只能轮询获取状态。团队已经实现了 WebSocket Gateway（TaskGateway），但队列处理器没有调用它。

**修复**：
- QueueModule 导入 WebSocketModule，使 TaskGateway 可注入
- QueueProcessor 注入 TaskGateway
- 任务开始时推送 `status: 'processing', progress: 10`
- 任务完成时推送 `status: 'completed', progress: 100, resultUrl`
- 任务失败时推送 `status: 'failed', errorMessage`
- 删除了重复的 `userId` 变量声明

**效果**：前端通过 WebSocket 实时收到任务进度，无需轮询。团队已有的 `useTaskProgress` Hook 现在能真正工作。

**涉及文件**：
- `apps/ai-video-api/src/modules/queue/queue.module.ts` - 导入 WebSocketModule
- `apps/ai-video-api/src/modules/queue/queue.processor.ts` - 注入 TaskGateway，三处推送

### 2.2 keyframe.service 越权风险修复 + 动态尺寸

**问题 1（安全漏洞）**：`verifyShotAccess` 只验证了 shot 是否属于 project，没有验证 project 是否属于当前 user。任何登录用户只要知道 projectId 就能操作别人的分镜首尾帧。

**修复**：`verifyShotAccess` 增加 `prisma.project.findFirst({ where: { id: projectId, userId } })` 校验，项目不属于该用户时抛出 NotFoundException。

**问题 2**：首尾帧图片尺寸硬编码为 `576x1024`（9:16），不适应 16:9 或 1:1 项目。

**修复**：新增 `getDimensionsByAspectRatio` 方法，根据项目的 `aspectRatio` 字段动态计算尺寸：
- 16:9 -> 1024x576
- 1:1 -> 768x768
- 9:16 -> 576x1024（默认）

**涉及文件**：
- `apps/ai-video-api/src/modules/storyboard/keyframe.service.ts`

---

### 2.3 script-import.service upsert bug 修复 + 软删除

**问题 1（Bug）**：`importToProject` 方法用 `prisma.storyboard.upsert({ where: { id: projectId } })` 查询 storyboard，但 storyboard 的 `id` 是 UUID，不是 projectId。这会导致 upsert 永远走 create 分支（或报错），每次导入都创建重复的 storyboard 记录。原代码注释 `// This won't work perfectly but we'll handle it` 也承认了这个问题。

**修复**：改为 `findFirst({ where: { projectId } })` 查询已有 storyboard，不存在时才 create。

**问题 2**：`deleteMany` 物理删除所有旧分镜，无法回溯之前的版本。

**修复**：改为将旧分镜标记为 `archived: true`（在 params JSON 中追加 archived 字段和 archivedAt 时间戳），与 storyboard.service 的归档逻辑一致。

**涉及文件**：
- `apps/ai-video-api/src/modules/storyboard/script-import.service.ts`

### 2.4 global-exception.filter 增强 Prisma 错误处理

**问题**：全局异常过滤器只处理了 `PrismaClientKnownRequestError` 和 `PrismaClientValidationError`，未处理：
- `PrismaClientUnknownRequestError` - 数据库未知错误，会走到 `Error` 分支泄漏内部错误信息
- `PrismaClientInitializationError` - 数据库连接失败，同样泄漏内部信息
- 生产环境下 `Error` 分支直接返回 `exception.message`，可能暴露堆栈或 SQL 细节

**修复**：
- 新增 `PrismaClientUnknownRequestError` 处理：返回 500 + "数据库未知错误，请稍后重试"
- 新增 `PrismaClientInitializationError` 处理：返回 503 + "数据库连接失败，请稍后重试"
- `Error` 分支增加生产环境保护：`NODE_ENV === 'production'` 时不返回原始错误信息

**涉及文件**：
- `apps/ai-video-api/src/common/filters/global-exception.filter.ts`

### 2.5 前端 API 客户端补全

**问题 1**：`charactersApi` 缺少 `generateViews` 和 `generateVariant` 方法。后端已有 `POST /:characterId/generate-views` 和 `POST /:characterId/variants/:variantType` 端点，但前端没有调用方法，角色四视图和变体生成功能在前端无法使用。

**修复**：在 `charactersApi` 中新增：
- `generateViews(projectId, characterId)` - 调用四视图生成
- `generateVariant(projectId, characterId, variantType)` - 调用变体生成

**问题 2**：`storyboardApi.generate` 的参数类型是 `{ prompt: string }`，但后端 `GenerateShotsDto` 期望 `{ story: string; style?: string; characterDescriptions?: string[] }`。参数名不匹配会导致后端收到 `undefined`。

**修复**：参数类型改为 `{ story: string; style?: string; characterDescriptions?: string[] }`。

**涉及文件**：
- `apps/ai-video-web/lib/api.ts`

## 五、第三轮优化（安全加固 + 功能补全）

> 日期：2026-07-20（续）
> 范围：越权修复、软删除统一、代码复用、功能补全

### 5.1 storyboard.controller 越权修复

**问题**：`listShots` 和 `deleteShot` 没有传 userId，任何登录用户知道 projectId 就能查看/删除别人的分镜。

**修复**：两个端点都加上 `@CurrentUser('id')` 参数并传递给 service。

**涉及文件**：`storyboard.controller.ts`、`storyboard.service.ts`

### 5.2 storyboard.service listShots 过滤 archived + deleteShot 软删除

**修复**：
- `listShots` 增加 userId 参数和 `verifyProjectAccess` 校验
- `deleteShot` 增加 userId 参数和项目归属校验
- `deleteShot` 从物理删除改为软删除（标记 `status: 'archived'` + `params.archived`）
- 新增 `verifyProjectAccess` 私有方法

### 5.3 generations.service 复用 resolveApiKey + 项目验证

**问题**：
- `resolveKeyInfo` 重复实现了 `ModelsService.resolveApiKey` 的逻辑
- `listTasks` 和 `getTask` 没有验证项目归属

**修复**：
- `createTask` 改用 `modelsService.resolveApiKey` 解析 key
- 新增 `findKeyId` 方法查找具体 keyId
- `listTasks` 和 `getTask` 增加 `verifyProjectAccess` 校验
- 删除冗余的 `resolveKeyInfo` 方法

**涉及文件**：`generations.service.ts`

### 5.4 projects.service 新增 updateProject

**问题**：项目只有创建和删除，缺少更新（改名、改描述、改比例、改状态）。

**修复**：
- 新增 `UpdateProjectDto`（name/description/style/aspectRatio/status 均可选）
- service 新增 `updateProject` 方法，先验证项目归属再更新
- controller 新增 `PUT /projects/:id` 端点
- 前端 `projectsApi` 新增 `updateProject` 方法

**涉及文件**：`projects.service.ts`、`projects.controller.ts`、`dto/update-project.dto.ts`（新增）、`lib/api.ts`

### 5.5 compose.service 过滤 archived 分镜

**问题**：合成时过滤条件只检查 `resultUrl`，已归档的分镜如果有 resultUrl 会被错误地包含进合成。

**修复**：过滤条件增加 `s.status !== 'archived'`。

**涉及文件**：`compose.service.ts`
