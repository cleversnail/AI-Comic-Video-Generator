# AI 漫剧生成器（PC Web）产品方案

> 版本：V1.1  
> 目标平台：PC 端 Web 产品  
> 核心定位：面向普通创作者、漫画/短剧爱好者的 AI 漫剧生成编排平台。通过“角色设计 → 故事编排 → 镜头分镜 → 视频生成”的引导式流程，连接用户自有 API Key 与多个第三方 AI 模型，降低 AI 视频创作门槛。  
> 关键特性：**各环节 AI 能力支持用户自主选择模型，并自行配置 API Key。**

---

## 一、产品定位与目标用户

### 1.1 产品定位
- **一句话定位**：输入角色和剧情，连接用户自选的 AI 模型，即可生成连续、有镜头感的 AI 漫剧视频。
- **核心价值**：
  - 保证角色一致性
  - 引导用户从零创作完整漫剧
  - 提供可操控的镜头语言与情绪节奏
  - **模型中立，用户自选**：各环节 AI 能力（角色图、分镜、视频、配音、音乐等）均支持用户自主选择模型，并自行配置 API Key
  - **创作编排中枢**：不只是单一模型工具，而是整合多个 AI 能力的创作工作流平台

### 1.2 目标用户
| 用户类型 | 需求 | 关注点 |
|---------|------|--------|
| 短视频/短剧创作者 | 快速生成原创漫剧内容 | 出片速度、角色统一、剧情连贯 |
| 小说/漫画爱好者 | 将文字剧情可视化 | 人物还原、画面精美、镜头感 |
| AI 绘画/视频尝鲜者 | 探索 AI 创作 | 易用性、效果可控、玩法新鲜 |
| 自媒体/网文作者 | 为作品制作宣传漫剧 | 分镜叙事、风格化、批量生成 |

## 二、核心创作流程

整体采用**引导式 Step-by-Step 流程**，降低创作门槛：

```
Step 1: 创建项目
    ↓
Step 2: 角色设计（角色提示词 + AI 生成角色图）
    ↓
Step 3: 故事编排（输入故事情节/剧本）
    ↓
Step 4: 分镜生成（AI 自动拆分为分镜，用户可编辑）
    ↓
Step 5: 镜头与提示词控制（引导用户补充镜头提示词）
    ↓
Step 6: 视频生成（按分镜生成视频片段）
    ↓
Step 7: 后期合成（剪辑、配音、配乐、字幕）
    ↓
Step 8: 导出/发布
```

## 三、功能模块设计

### 3.1 项目管理模块
| 功能 | 说明 |
|------|------|
| 新建项目 | 输入漫剧名称、选择风格（现代/古风/赛博/校园/悬疑等）、选择视频比例（16:9 / 9:16 / 1:1） |
| 项目列表 | 查看历史项目、草稿、已生成作品 |
| 项目设置 | 修改项目名称、风格、比例、分辨率、目标时长 |
| 自动保存 | 每个步骤自动保存，支持版本回溯 |

### 3.2 角色设计模块
| 功能 | 说明 |
|------|------|
| 角色创建 | 用户输入角色名、性别、年龄、身份、外貌、服装、性格标签 |
| 角色提示词生成器 | 根据结构化输入自动生成英文/中文角色提示词（可由用户选择的大语言模型处理） |
| AI 生成角色图 | 基于角色提示词，调用用户已配置的图像生成模型生成多张候选角色图 |
| 模型选择 | 在角色生成环节，用户可切换使用 FLUX、Midjourney、Stable Diffusion、即梦、可灵、通义万相等模型 |
| 角色图选择 | 用户选择最满意的一张作为主形象 |
| 角色一致性锁定 | 将选定角色图作为参考图，后续生成统一画风/角色形象 |
| 多角色管理 | 支持一部漫剧创建多个角色，建立人物关系 |
| 角色变体 | 支持生成角色的不同表情、角度、服装变体 |

#### 角色提示词引导示例
```
【基础信息】
- 姓名：林小夏
- 性别：女
- 年龄：18
- 身份：高中生

【外貌特征】
- 发型：黑色中长直发，齐刘海
- 眼睛：棕色大眼睛
- 服饰：蓝白校服，红色领结，背着米色书包

【性格/气质】
- 温柔、内向、略带忧郁

【风格】
- 日式少女漫画风格，柔和光影，电影感

AI 自动生成的提示词：
"A 18-year-old Japanese anime girl, long straight black hair with bangs, large brown eyes, gentle and melancholic expression, wearing blue and white school uniform with red ribbon, beige backpack, soft lighting, cinematic composition, shoujo manga style, high quality, detailed face --ar 9:16"
```

### 3.3 故事编排模块
| 功能 | 说明 |
|------|------|
| 剧情输入 | 用户输入完整故事情节或分集剧本 |
| 剧情模板 | 提供常见剧情模板（相遇、冲突、反转、救赎、悬疑等） |
| AI 剧情润色 | 调用用户选择的大语言模型，对用户提供的故事进行扩展/精简/分镜化处理 |
| 章节/剧集管理 | 支持多集连续创作，每集独立分镜 |
| 角色绑定 | 将故事中出现的角色与已创建角色关联 |

#### 故事输入示例引导
```
请用简洁的语言描述你的故事情节：

示例：
林小夏放学后在空荡的教室独自发呆，窗外的夕阳洒在她的脸上。
突然，门被推开，转学生顾言走了进来，手里拿着一本她丢失的日记本。
他温柔地递还给她，两人目光相遇，林小夏的脸微微泛红。
```

### 3.4 分镜生成模块
| 功能 | 说明 |
|------|------|
| 自动分镜 | 调用用户选择的大语言模型，将故事情节拆分为多个分镜，每镜包含画面描述、出镜角色、场景、情绪 |
| 分镜编辑 | 用户可增删分镜、调整顺序、修改画面描述 |
| 分镜预览 | 每个分镜生成静态预览图 |
| 角色与场景一致性 | 自动根据分镜内容匹配合适角色和场景风格 |
| 分镜时长设置 | 设置每个镜头的时长（默认 3-5 秒） |

### 3.5 镜头控制与提示词引导模块
这是本产品的关键差异化能力：**引导用户用自然语言控制镜头语言**。

#### 镜头提示词结构
每个分镜的提示词由以下部分组成：

```
[画面主体] + [角色动作/表情] + [场景/环境] + [镜头语言] + [光影/色调] + [情绪/氛围] + [风格]
```

#### 镜头语言引导词库
| 类别 | 可选关键词 |
|------|------------|
| 景别 | 特写 Close-up、近景 Medium shot、中景、全景 Wide shot、远景 Extreme long shot |
| 角度 | 平视、俯视、仰视、鸟瞰、主观视角 POV、过肩镜头 OTS |
| 运镜 | 推镜 Push in、拉镜 Pull out、摇镜 Pan、跟拍 Follow、环绕 Orbit、固定 Static |
| 景深 | 浅景深 Shallow depth of field、深景深 Deep focus、背景虚化 Bokeh |
| 光影 | 逆光 Backlight、侧光 Side light、顶光 Top light、柔光 Soft light、硬光 Hard light、黄昏 Golden hour |
| 色调 | 暖色调 Warm tone、冷色调 Cool tone、高对比 High contrast、低饱和 Desaturated、电影感 Cinematic |
| 情绪 | 紧张 Tense、温馨 Warm、悲伤 Sad、浪漫 Romantic、神秘 Mysterious、欢快 Cheerful |

#### 镜头提示词示例
```
分镜 1：
- 景别：特写
- 角度：平视，略带仰视
- 运镜：缓慢推镜
- 描述：林小夏的眼睛，眼眶微微泛红，夕阳反光映在瞳孔中，表情从平静到动容
- 风格：柔和光影，电影感，少女漫风格

AI 生成完整提示词：
"Close-up shot, slow push-in, eye-level angle of Lin Xiaoxia, a teenage anime girl with brown eyes, tears forming in her eyes, golden sunset reflection in pupils, expression shifting from calm to emotional, soft lighting, cinematic bokeh, shoujo manga style, warm tone --ar 9:16"
```

#### 提示词智能辅助
- **关键词推荐**：根据当前分镜内容推荐镜头词
- **一键优化**：调用用户选择的大语言模型，将口语化描述转为标准视频生成提示词
- **角色锁定注入**：自动在提示词中加入角色一致性参考（如 LoRA、Reference Image 等）
- **负面提示词**：自动生成负面提示词（畸形、多手指、画面抖动等）

### 3.6 视频生成模块
| 功能 | 说明 |
|------|------|
| 单镜生成 | 调用用户选择的视频生成模型，按分镜逐个生成视频片段 |
| 批量生成 | 一键生成所有分镜视频，支持任务队列 |
| 模型选择 | 用户可为视频生成选择可灵、豆包 PixelDance/Seaweed、Pika、Runway、Luma、CogVideo、Wan 等模型 |
| 角色一致性保持 | 使用角色参考图、IPAdapter、LoRA 等技术保证角色一致（具体能力取决于所选模型） |
| 首帧/尾帧控制 | 可设置关键帧，保证镜头衔接自然（需所选模型支持） |
| 生成参数设置 | 选择模型、步数、CFG、分辨率、运动幅度、时长 |
| 重新生成 | 对不满意的分镜单独重新生成，可切换模型重试 |

### 3.7 后期合成模块
| 功能 | 说明 |
|------|------|
| 时间线编辑 | 拖拽式排列分镜顺序，调整时长 |
| 转场效果 | 提供淡入淡出、闪白、叠化、划像等转场 |
| 配音/旁白 | 调用用户选择的 TTS 模型，输入文字生成 AI 配音，选择音色、语速、情绪 |
| 字幕生成 | 根据剧情或配音自动生成字幕，支持样式调整 |
| 背景音乐 | 调用用户选择的音乐生成模型，按情绪推荐/生成背景音乐，支持音量调节 |
| 音效库 | 调用用户选择的音效模型或内置素材库，提供常用音效（开门、脚步声、风声、雨声等） |
| 预览导出 | 实时预览完整漫剧，导出 MP4/WebM |

### 3.9 模型与 API Key 管理模块（新增）

本产品的关键机制：**用户自主选择每个创作环节使用的 AI 模型，并自行配置对应平台的 API Key**。产品本身不绑定任何单一模型，而是作为中立的创作编排中枢。

#### 支持的 AI 能力环节与可选模型

| 创作环节 | 说明 | 可选模型/服务示例 |
|---------|------|-----------------|
| 大语言模型（LLM） | 用于剧情润色、分镜拆分、提示词优化 | OpenAI GPT-4 / GPT-3.5、Claude 3、Gemini、文心一言、通义千问、Kimi、智谱 GLM、DeepSeek 等 |
| 角色图片生成 | 根据角色提示词生成角色图 | Midjourney、Stable Diffusion（在线/自部署）、FLUX、即梦、可灵、LiblibAI、通义万相 等 |
| 视频生成 | 根据分镜提示词生成视频片段 | 可灵 Kling、豆包即梦（PixelDance/Seaweed）、Pika、Runway Gen-3、Luma Dream Machine、CogVideo、Wan 2.1、Hailuo MiniMax 等 |
| 配音/TTS | 将文字转为角色配音 | ElevenLabs、Azure TTS、讯飞配音、豆包语音、MiniMax TTS、Fish Audio 等 |
| 音乐生成 | 根据情绪生成背景音乐 | Suno、Udio、Stable Audio、天工音乐、Mureka 等 |
| 音效生成 | 生成/匹配环境音效 | ElevenLabs Sound Effects、Stable Audio、剪映/CapCut 音效库 等 |

#### 核心功能

| 功能 | 说明 |
|------|------|
| 模型商店 | 展示各创作环节支持的模型列表，包含模型能力简介、适用场景、官方文档链接、计费方式说明 |
| API Key 配置 | 用户为每个模型单独配置 API Key，支持多个 Key（如主 Key/备用 Key） |
| Key 安全存储 | API Key 存储于用户本地浏览器或后端加密存储，服务端调用时不落日志 |
| 环节级模型选择 | 在角色生成、视频生成、配音等每个环节，用户可切换使用的模型 |
| 项目级默认模型 | 用户可设置每个项目的默认模型组合，新项目可继承 |
| 模型参数自定义 | 用户可调整温度、CFG、步数、分辨率、运动幅度等参数 |
| 任务路由 | 系统将用户任务分发到对应模型的 API |
| 用量统计 | 统计各模型的调用次数、Token 消耗、费用估算 |
| 失败重试/切换 | 某个模型失败时，可自动重试或提示切换到备用模型 |

#### API Key 配置流程

```text
用户进入「模型设置」
    │
    ▼
选择创作环节（如：视频生成）
    │
    ▼
浏览可选模型列表
    │
    ▼
选择模型（如：可灵 Kling）
    │
    ▼
点击「去官方获取 API Key」跳转官方平台
    │
    ▼
复制 API Key 回到本产品
    │
    ▼
粘贴 API Key → 点击验证
    │
    ▼
验证通过，保存配置
    │
    ▼
该创作环节默认使用此模型
```

#### 用户使用场景示例

```
场景 1：
- 角色图生成：用 FLUX（本地 Stable Diffusion）+ 自己的 Replicate API Key
- 分镜拆分：用 DeepSeek-V3
- 视频生成：用可灵 Kling Pro
- 配音：用 ElevenLabs

场景 2：
- 角色图生成：用即梦
- 分镜拆分：用 Kimi
- 视频生成：用豆包 PixelDance
- 配音：用豆包语音

场景 3：
- 全部环节都用 OpenAI 生态，追求英文提示词效果最好
```

#### 产品价值

1. **避免绑定单一模型**：用户可根据效果、价格、速度自由选择
2. **成本透明**：用户直接对接各模型官方计费，产品可加收少量平台服务费或纯免费
3. **紧跟技术发展**：新模型出现时只需扩展模型商店，不需要重构产品
4. **高级用户友好**：让有自建模型或企业 API 资源的用户能充分利用自己的 Key
5. **合规与数据安全**：用户数据通过用户自己的 Key 调用，降低平台合规压力
| 功能 | 说明 |
|------|------|
| 我的角色库 | 保存常用角色，跨项目复用 |
| 我的场景库 | 保存常用场景描述/风格 |
| 提示词模板库 | 收藏常用的镜头提示词模板 |
| 风格预设 | 漫画风、写实风、3D、水墨、像素等 |

## 五、提示词引导体系设计

为降低用户写提示词的难度，产品设计三层引导：

### 5.1 第一层：结构化表单（适合新手）
用户只需填写表单，系统自动拼接提示词：
- 选择分镜
- 选择景别/角度/运镜/光影/情绪（下拉框）
- 输入画面主体动作
- 系统生成完整提示词

### 5.2 第二层：自然语言辅助（适合进阶用户）
用户输入口语化描述，AI 自动优化为标准提示词：
- 输入："我想要一个女生在夕阳下回头看的镜头"
- 输出："Medium shot, golden hour backlight, a teenage anime girl turning her head slowly, hair glowing in sunset, warm tone, cinematic composition, shoujo manga style --ar 9:16"

### 5.3 第三层：专家模式（适合高级用户）
直接编辑完整提示词，可插入：
- 参考图链接
- LoRA 权重
- 自定义负面提示词
- ControlNet 参数

## 六、技术实现要点（初步）

| 模块 | 关键技术 |
|------|----------|
| 角色生成 | Stable Diffusion / FLUX + LoRA / IP-Adapter |

| 后期合成 | FFmpeg / WebCodecs + Web Audio API |
| 前端 | React / Vue + 可视化时间线组件 |
| 后端 | 任务队列（Celery / BullMQ）管理视频生成 |

| 模式 | 说明 |
|------|------|

## 八、后续迭代方向

1. **AI 剧本生成**：输入关键词/梗概，调用用户选择的大语言模型自动生成完整剧本
2. **更多模型接入**：持续扩展模型商店，支持最新的图像/视频/TTS/音乐模型
3. **自定义模型接入**：允许企业/高级用户接入私有 API 或本地部署模型
4. **角色对话配音**：支持多角色对话配音与口型同步
5. **社区/模板市场**：用户分享角色、剧本、漫剧模板
6. **多语言支持**：支持中文、英文、日文等多语言创作
7. **AI 音乐生成**：根据剧情情绪自动生成配乐
8. **IP 品牌化**：支持固定 IP 形象长期系列化创作
9. **批量生成剧集**：一次生成多集连载内容

## 九、总结

AI 漫剧生成器的核心体验在于：

> **让没有专业镜头语言知识的普通用户，也能通过结构化引导，创作出角色统一、剧情连贯、有电影感的 AI 漫剧视频。**

关键成功要素：
1. **模型中立与自由组合**：让用户能根据效果、成本、速度选择最适合自己的模型组合
2. **角色一致性**：解决 AI 视频角色漂移痛点
3. **引导式创作**：降低从 0 到 1 的创作门槛
4. **镜头语言可控**：让用户能轻松控制画面叙事
5. **端到端工作流**：从角色、剧本、分镜、视频到后期一站式完成
6. **安全透明的 API Key 机制**：让用户放心使用自有 Key，平台不过度控制
