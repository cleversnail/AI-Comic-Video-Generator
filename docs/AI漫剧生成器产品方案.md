# AI 漫剧创作台（PC Web）产品方案

> 版本：V2.0  
> 目标平台：PC 端 Web 产品  
> 核心定位：面向普通创作者、短视频作者、漫画/网文爱好者的 AI 漫剧创作工作台。通过「角色设定 → 剧情输入 → 分镜编排 → AI 视频生成 → 导出」的流程，将文字故事转化为角色一致、镜头连贯、有漫剧风格的短视频。  
> 关键特性：**模型中立、用户自带 API Key、极速模式优先、专业模式后置。**

---

## 一、产品定位与目标用户

### 1.1 产品定位
- **一句话定位**：输入角色和故事，3 分钟得到第一条漫剧草稿；想精修，随时进入专业工作台。
- **核心价值**：
  - **极速出片**：新手 3-4 步即可看到第一个可播放的漫剧草稿
  - **角色一致性**：跨分镜、跨章节保持角色形象稳定
  - **镜头语言引导**：不用写复杂提示词，选择参数即可控制画面
  - **模型中立**：每个创作环节都支持用户选择自己的模型和 API Key
  - **渐进式深度**：从极速模式平滑过渡到专业分镜控制

### 1.2 产品名称建议
- 主推荐：**AI 漫剧创作台**
- 原因：相比"生成器"，"创作台"更符合多步骤、多模型、多素材的工作流定位

### 1.3 目标用户
| 用户类型 | 需求 | 关注点 |
|---------|------|--------|
| 短视频/短剧创作者 | 快速生成原创漫剧内容 | 出片速度、角色统一、剧情连贯 |
| 小说/漫画爱好者 | 将文字剧情可视化 | 人物还原、画面精美、镜头感 |
| AI 绘画/视频尝鲜者 | 探索 AI 创作 | 易用性、效果可控、玩法新鲜 |
| 自媒体/网文作者 | 为作品制作宣传漫剧 | 分镜叙事、风格化、批量生成 |

## 二、双模式核心创作流程

### 2.1 极速模式（默认入口）
目标：让用户在 3 分钟内看到第一个粗剪作品，建立成就感。

```
Step 1: 创建/选择角色
    - 输入角色名、外貌、性格
    - 或从角色模板中选择
    - 生成或上传角色参考图
    ↓
Step 2: 输入故事
    - 输入一段剧情、小说片段、短剧梗概
    - 选择风格：校园 / 古风 / 悬疑 / 恋爱 / 热血 / 赛博等
    ↓
Step 3: 一键生成粗剪
    - AI 自动拆分分镜
    - 自动生成静态预览图
    - 自动分配镜头语言
    - 自动生成字幕/旁白草稿
    - 自动调用视频模型生成短片段
    ↓
Step 4: 预览与精修
    - 观看粗剪结果
    - 不满意的分镜可单独重写、重画、重生成
    - 可切换到专业模式继续编辑
```

### 2.2 专业模式（进阶精修）
目标：给认真创作的用户提供可控、可回退、可复用的完整工作流。

```
1. 项目初始化（最少信息，其他配置后置）
   ↓
2. 角色与角色变体管理
   ↓
3. 故事 / 剧本 / 台词编排
   ↓
4. 分镜 + 镜头控制一体化编辑
   ↓
5. 静态预览 + 故事板阅读预览
   ↓
6. 音频 / 字幕 / 时间轴预排
   ↓
7. 视频片段生成
   ↓
8. 后期合成与导出
```

### 2.3 与原方案的关键变化

| 原方案 | V2.0 优化 |
|---|---|
| 线性 8 步向导 | 双模式：极速模式 4 步 + 专业模式检查表式导航 |
| 分镜生成和镜头控制分两页 | 合并为一体化故事板编辑器，卡片展开编辑镜头参数 |
| 视频生成后再做配音/字幕 | 先确定台词、旁白、字幕、时长，再生成视频 |
| 创建项目时填写大量配置 | 只要求项目名称，风格和比例后置 |
| 单张参考图锁定角色 | 角色变体集合 + 锁定强度（宽松/中等/严格） |
| 5 个提示词入口 | 统一为"创作助手 AI Assistant"，按上下文自动选择交互模式 |

### 3.2 角色设计与一致性模块

角色一致性是本产品的核心护城河之一，从"单张参考图"升级为"角色资产系统"。

#### 角色结构
```
角色 Character
├─ 基础信息（姓名、年龄、性别、身份、性格）
├─ 主形象 Main Identity
│  ├─ 主参考图
│  ├─ 标准服装
│  ├─ 标准发型
│  └─ 标准脸部特征
├─ 角色变体 Variants
│  ├─ 表情变体：开心 / 哭泣 / 愤怒 / 惊讶
│  ├─ 服装变体：校服 / 睡衣 / 古装 / 战斗服
│  ├─ 角度变体：正面 / 侧面 / 背影 / 半身
│  └─ 场景变体：白天 / 夜晚 / 雨天 / 战斗状态
└─ 模型适配信息
   ├─ 支持参考图的模型：传参考图
   ├─ 支持角色引用的模型：传 character ref / cref
   ├─ 支持 LoRA 的模型：绑定 LoRA
   └─ 不支持参考图的模型：注入角色提示词
```

#### 角色锁定强度
| 锁定强度 | 适用场景 | 系统行为 |
|---|---|---|
| 宽松 | 创意发散、不同画风尝试 | 主要注入文字描述，允许造型变化 |
| 中等 | 常规漫剧生成 | 使用参考图 + 角色提示词，兼顾一致性和画面自由度 |
| 严格 | 连载角色、品牌 IP | 强制参考主形象/变体，限制发型、服装、脸部变化 |

#### 功能清单
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 创建角色 | 输入角色名、性别、年龄、身份、外貌、服装、性格 | P0 |
| 角色提示词生成 | 由创作助手根据描述自动生成角色提示词 | P0 |
| 生成/上传主参考图 | 调用用户配置的图像模型生成，或上传已有图片 | P0 |
| 角色绑定到分镜 | 分镜中自动识别或手动指定出镜角色 | P0 |
| 角色信息自动注入 | 生成提示词中自动注入角色描述 | P0 |
| 角色变体集合 | 表情、服装、角度、场景变体管理 | P1 |
| 角色锁定强度 | 宽松/中等/严格三档 | P1 |
| 跨集角色复用 | 同一角色在多个项目中复用 | P1 |
| LoRA / FaceID 深度绑定 | 高级角色一致性方案 | P2 |

#### 创作助手行为示例
| 当前页面 | 用户输入 | 助手行为 |

### 3.5 音频、字幕与时间轴模块

时间轴前置到视频生成之前，先确定节奏再生成视频。

```

  ↓
台词 / 旁白 / 字幕
  ↓
每个分镜预估时长
  ↓
音频生成或占位
  ↓
视频生成按分镜时长对齐
  ↓
后期合成微调
```

### 3.8 模型与 API Key 管理模块

#### 设计原则
1. **模型商店和 Key 配置合并**：在模型详情页直接配置 Key
2. **默认只暴露项目级配置**：全局默认作为后台兜底
3. **任务级切换属于高级功能**：P1/P2 再暴露

#### 推荐交互
```
用户在视频生成页选择"可灵 Kling"
    ↓
如果未配置 Key
    ↓
弹出模型详情 / Key 配置面板
    ↓
展示：模型介绍、适用场景、价格说明、官方获取 Key 链接
    ↓
用户粘贴 Key，点击验证
    ↓
验证通过后直接用于当前项目
```

#### 配置层级
```
任务级配置 > 项目级配置 > 全局默认配置
```

UI 暴露策略：
- P0：只暴露项目级配置
- P1：高级设置中暴露全局默认
- P2：单任务弹窗中暴露任务级临时切换

#### 功能清单
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 模型列表 | 按能力环节展示可选模型 | P0 |
| 模型详情 | 能力、场景、文档、价格说明，内嵌 Key 配置 | P0 |
| API Key 添加/验证/删除 | 在同一页面完成 | P0 |
| Key 加密存储 | 服务端加密，不落日志 | P0 |
| 项目级默认模型配置 | 为当前项目选择默认模型 | P0 |
| 多 Key 主备切换 | 同一模型多个 Key | P1 |
| 用量统计 | 调用次数、费用估算 | P2 |

#### 首批模型建议
| 能力 | 建议首批 | 原因 |
|---|---|---|
| LLM | DeepSeek、OpenAI Compatible | 中文成本低，兼容生态广 |
| 图像 | Replicate FLUX、通义万相/即梦 | 一个国际 API，一个国内 API |
| 视频 | 可灵、豆包/即梦 | 国内用户可用性强 |
| TTS | ElevenLabs、豆包语音/MiniMax | 覆盖英文和中文 |
| 合成 | FFmpeg | 稳定、成熟、可控 |

## 五、提示词引导体系

统一为**创作助手 AI Assistant**，根据用户所在上下文自动选择最适合的交互模式。

| 模式 | 触发方式 | 适用场景 |
|---|---|---|
| 结构化表单 | 用户在分镜卡片中点击参数 | 选择景别、角度、运镜等 |
| 自然语言辅助 | 用户在任意输入框描述意图 | "想要更浪漫一点" |
| 专家模式 | 高级用户展开完整提示词编辑 | 直接编辑完整 prompt |

创作助手始终可见，用户随时可以向它描述需求，系统根据当前页面自动判断任务类型。

## 六、技术实现要点

| 模块 | 关键技术 |
|------|----------|
| 角色生成 | Stable Diffusion / FLUX + IP-Adapter / Redux / LoRA |
| 角色一致性 | 分层策略：参考图注入 / 提示词注入 / 模型自身能力 |

| 配音 | TTS Adapter（ElevenLabs / 豆包语音 / MiniMax） |
| 后期合成 | FFmpeg |
| 前端 | Next.js 14 + React + TypeScript + Tailwind CSS |
| 后端 | NestJS + Prisma + MySQL + Redis + BullMQ |
| 对象存储 | MinIO / S3 / OSS |

### Adapter 拆分
不按单一 `ModelAdapter` 设计，而按能力拆分：
- `LLMAdapter`
- `ImageAdapter`
- `VideoAdapter`
- `TTSAdapter`
- `MusicAdapter`
- `SoundAdapter`

上层通过 `adapterFactory.getAdapter(capability, modelId)` 路由。

### 文件链路区分
- **素材链路**：用户上传 → 前端直传对象存储 → 后端记录
- **产物链路**：后端创建任务 → Worker 调用模型 → 下载/回调写入对象存储 → 更新任务状态

## 七、UI/UX 关键原则

### 7.1 视觉方向
- 暗色影院感 + 漫画分镜网格 + 可控参数面板
- 视频预览区保持影院黑
- 文本编辑区（故事、提示词、字幕）使用中灰底，降低视觉疲劳

### 7.2 导航方式
从线性步骤导航改为**创作检查表导航**：
- 未开始 / 进行中 / 已完成 / 需要处理 / 生成失败
- 用户可在已完成步骤间自由跳转

### 7.3 核心页面收敛
| 页面 | 路径 | 说明 |
|------|------|------|
| 首页 | `/` | 产品介绍、示例、开始创作 |
| 项目列表 | `/projects` | 最近创作、草稿、成片 |
| 创作工作台 | `/projects/:id/studio` | 角色、故事、分镜、预览的主工作区 |
| 生成队列 | `/projects/:id/generate` | 视频任务、重试、结果管理 |
| 后期导出 | `/projects/:id/export` | 预览、字幕、音频、导出 |
| 模型中心 | `/settings/models` | 模型商店 + Key 管理 |

## 十、关键成功要素

1. **极速模式优先**：让用户 3 分钟内看到第一个作品
2. **分镜静态预览闭环**：生成视频前先看构图
3. **角色一致性分层落地**：参考图 + 变体 + 锁定强度
4. **音频/字幕时间轴前置**：避免音画不同步返工
5. **模型商店与 Key 配置合并**：降低配置门槛
6. **Adapter 按能力拆分**：支持多模型灵活扩展
7. **素材与产物链路分离**：明确文件流方向
8. **安全透明的 API Key 机制**：建立用户信任
