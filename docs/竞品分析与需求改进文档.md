## 二、竞品产品定位深度分析

### 2.1 printfilm（AI 漫剧工场）

**一句话定位**：面向短剧/漫剧/动态漫画创作者的 AI 生产工作台，Script-to-Asset-to-Keyframe 工作流。

**核心理念 — 关键帧驱动**：
1. 先画后动：先生成镜头的起始帧与结束帧
2. 插值生成：通过视频模型在关键帧之间生成运动过渡
3. 资产约束：画面生成参考角色定妆、服装变体和场景概念图
4. 流程化生产：每个阶段围绕同一项目状态推进

**四阶段流程**：
```
Phase 01: 剧情创作 → 输入故事想法，AI 生成剧本和分镜
Phase 02: 场景角色 → 生成角色设计、服装变体、场景概念图
Phase 03: AI 工作台 → 逐镜头生成关键帧和视频片段
Phase 04: 制片导出 → 预览、导出关键帧/视频/项目资产
```

**值得借鉴**：
- ✅ 「关键帧驱动」思路 — 先确定首尾帧再生成视频，比直接 Text-to-Video 更可控
- ✅ 资产管理模块 — 统一管理提示词、角色、场景资产
- ✅ 桌面端 + Web 双端覆盖

### 2.2 AIComicBuilder

**一句话定位**：AI 驱动的漫剧生成器，从剧本到动画视频的全自动流水线。

**生成流水线**：
```
剧本输入 → 剧本解析 → 角色提取 → 角色四视图
                                      ↓
                                   智能分镜
                                      ↓
                         参考帧生成 / 首尾帧生成（逐镜头）
                                      ↓
                              视频提示词生成（逐镜头）
                                      ↓
                              视频生成（逐镜头）
                                      ↓
                              视频合成（FFmpeg）
```

**核心特性**：
- **角色四视图**：正面/四分之三/侧面/背面，确保一致性
- **风格自适应**：自动识别剧本风格（动漫/写实等），生成匹配风格
- **多 Agent 平台支持**：Dify、Coze、百炼三种 Agent 平台可选
- **分镜工作流**：编辑抽屉、角色内联面板、看板视图三种协作视图
- **帧图管理**：支持手动上传替换及一键清除

**值得借鉴**：
- ✅ 角色四视图 — 比单张参考图更能保证一致性
- ✅ 多 Agent 平台适配 — 不绑定单一编排引擎
- ✅ 风格自适应 — 自动识别并匹配风格
- ✅ 剧本导入（TXT/DOCX/PDF）— 降低输入门槛

---

### 2.3 openOii

**一句话定位**：以 LangGraph 为核心的 AI 漫剧多智能体协作学习项目。

**多 Agent 架构**：
```
orchestrator（编排者）
  ├── outline（大纲生成）
  ├── plan（计划生成）
  ├── scriptwriter（编剧）
  ├── critic（批评家/质检）
  ├── render（渲染/图像生成）
  ├── compose（合成）
  └── review（审核）
```

**核心特性**：
- **LangGraph 编排**：多阶段可恢复、可取消的 run 流程
- **WebSocket 实时进度**：前端实时展示生成进度
- **tldraw 无限画布**：在画布上审阅角色、分镜与结果
- **前端配置面板**：在线管理模型与基础服务配置

**值得借鉴**：
- ✅ 多 Agent 分工明确 — 每个 Agent 职责单一，便于独立优化
- ✅ 可恢复/可取消执行 — 任务中断后可从断点恢复
- ✅ 无限画布审阅 — 分镜审阅体验更直观

### 2.4 wind-comic

**一句话定位**：一句话进、整片短剧出的多 Agent AI 创作工作室。

**版本迭代历程**（最值得研究）：
```
v3  → 基础流水线
v6  → 生产工作室：角色工作台、提示词 IDE、长篇小说分集、风格画廊、导演控制台
v7  → 平台加固：DeepSeek v4 Pro + MiniMax 兜底、Polish Studio Pro 剧本审计
v8  → AI 导演站 + 高端设计：逐镜头摄影控制台、一致性锁定、情绪节奏曲线
v9  → Postgres 平台 + 变现：SQLite⇄Postgres 双驱动、多平台分发包、AAF 导出
v10 → 口型同步 + 模板市场 + 成本归因
```

**关键差异化能力**：
1. **Character Studio**：8 字段 DNA 身份锁定、三视图定妆照、跨项目 Cameo IP
2. **Polish Studio Pro**：行业级剧本审计 — AIGC 就绪评分、Save-the-Cat 三幕节拍检测
3. **口型同步**：viseme 关键帧轨道、嘴部-音频对齐评分、漂移自动校正
4. **模板市场**：保存→评分/收藏→一键复刻（含配音）
5. **成本可观测**：按阶段成本归因 + 预算守卫
6. **四维发布门控**：画面·一致性·口型·实测对齐
7. **多平台分发包**：抖音/快手/视频号/小红书/YouTube Shorts/B站

**值得借鉴**：
- ✅ 模板市场 — 爆款可复用，降低新用户门槛
- ✅ 成本可观测 — 用户清楚每一分钱花在哪里
- ✅ 口型同步 — 漫剧质量的关键差异化
- ✅ 剧本审计 — AI 自检剧本质量
- ✅ 多平台分发 — 一键生成多平台适配包
- ✅ 迭代速度 — v6→v10 快速迭代，每次都有核心功能

## 四、需求改进方案

### 4.1 P0 — 核心链路补齐（预计 2-3 周）

#### 4.1.1 关键帧驱动模式

**目标**：引入首尾帧生成→视频插值的工作流，提升视频可控性。

**需求描述**：
- 分镜生成后，为每个镜头生成「起始帧」和「结束帧」两张静态图
- 用户可在工作台中手动替换/调整首尾帧
- 视频生成时，传入首尾帧作为参考，由视频模型插值生成中间运动
- 支持「场景参考帧模式」：只提供一张参考图，AI 自动推断运动

**技术方案**：
- 前端：分镜卡片增加「首帧」「尾帧」两个缩略图位
- 后端：新增 `FirstLastFrameAdapter`，封装首尾帧逻辑
- 视频模型：可灵/Seedance 均支持首尾帧输入模式

**验收标准**：
- 用户可为每个分镜生成/上传首尾帧
- 视频生成任务可携带首尾帧参数
- 生成视频的首帧/尾帧与输入一致

---

#### 4.1.2 角色一致性升级

**目标**：从「单张参考图+提示词」升级为「角色资产系统」。

**需求描述**：
- 角色模型扩展：增加「主形象」「变体集合」「模型适配信息」
- 角色四视图生成：正面/四分之三/侧面/背面
- 角色变体管理：表情、服装、角度、场景变体
- 角色锁定强度：宽松/中等/严格三档
- 分镜自动绑定角色 + 生成时自动注入角色描述

**数据模型变更**：
```prisma
model Character {
  id          String   @id
  projectId   String
  name        String
  gender      String?
  age         String?
  personality String?
  appearance  String?   // 外貌描述
  mainImage   String?   // 主形象参考图
  viewImages  Json?     // 四视图 {front, three_quarter, side, back}
  variants    Json?     // 变体集合
  lockLevel   String?   // loose | medium | strict
  promptText  String?   // 角色提示词（AI 生成）
  // ...
}
```

**验收标准**：
- 创建角色时可生成四视图
- 分镜生成时自动注入角色描述和参考图
- 视频生成时角色形象一致

---

#### 4.1.3 剧本导入功能

**目标**：支持 TXT/DOCX/PDF 文件导入，AI 自动解析为结构化剧本。

**需求描述**：
- 支持上传 TXT、DOCX、PDF 格式文件
- AI 自动解析文本、提取角色、分集
- 解析结果可视化展示，用户可手动调整
- 支持长篇小说自动分集（按章节/按目标时长）

**技术方案**：
- 文件解析：mammoth (DOCX) + pdf-parse (PDF)
- AI 解析：LLM 提取角色列表、对话、场景描述
- 分集逻辑：按章节标题或指定长度自动拆分

---

### 4.2 P1 — 质量与效率提升（预计 3-4 周）

#### 4.2.1 剧本质量审计

**目标**：AI 自动检测剧本质量，提供改进建议。

**需求描述**：
- AIGC 就绪评分（0-100）
- 三幕结构检测（设置/对抗/解决）
- 对话质量检测（避免直白说教）
- 戏剧冲突强度分析
- 角色一致性检查

**参考实现**：wind-comic 的 Polish Studio Pro

---

#### 4.2.2 多 Agent 编排架构

**目标**：将单一 LLM 调用拆分为多 Agent 协作。

**需求描述**：
- 引入 Agent 编排层（LangGraph 或自研）
- Agent 分工：编剧 Agent、分镜 Agent、质检 Agent、导演 Agent
- 每个 Agent 可独立配置模型和参数
- 支持任务中断恢复和取消

**技术方案**：
- 后端新增 `AgentOrchestrator` 服务
- 每个 Agent 实现统一接口 `IAgent.execute(context)`
- 使用 Redis 存储中间状态，支持断点恢复

---

#### 4.2.3 长篇小说分集管理

**目标**：支持长篇小说/连续剧式多集创作。

**需求描述**：
- 项目级分集列表
- 角色按集关联，支持跨集复用
- 按章节自动分集或手动创建
- 分集独立预览和导出

---

#### 4.2.4 成本可观测

**目标**：让用户清楚每一步的 AI 调用成本。

**需求描述**：
- 按阶段成本归因：LLM / 图像 / 视频 / TTS 各占多少
- 项目级成本统计
- 预算守卫：设置预算上限，超出提醒
- 省钱建议：哪些步骤可以用更便宜的模型替代

---

### 4.3 P2 — 差异化竞争力（预计 4-6 周）

#### 4.3.1 口型同步

**目标**：生成的视频中角色嘴型与配音匹配。

**需求描述**：
- TTS 生成后，提取 viseme 关键帧
- 将 viseme 数据传入视频生成模型
- 生成后自动检测嘴部-音频对齐度
- 对齐度不足的镜头自动重新生成

---

#### 4.3.2 模板市场

**目标**：爆款项目可保存为模板，一键复用。

**需求描述**：
- 保存项目为模板（含风格、角色、节奏、配音）
- 模板预览（生成 5-10 秒预览片段）
- 模板评分与收藏
- 一键复刻（从模板创建新项目，自动填充所有设置）

---

#### 4.3.3 多平台分发包

**目标**：一键生成多平台适配的视频包。

**需求描述**：
- 支持平台：抖音、快手、视频号、小红书、YouTube Shorts、B站
- 自动适配各平台的分辨率、时长、比例要求
- 自动生成各平台的封面和标题建议

---

#### 4.3.4 导演控制台

**目标**：逐镜头的摄影参数精细控制。

**需求描述**：
- 每个分镜可配置：景别、机位、镜头、运镜、焦点
- 情绪/节奏曲线编辑
- 连续性检查（相邻镜头的构图/光线一致性）
- Seed 锁定（同一镜头多次生成保持一致性）

---

## 五、技术架构改进建议

### 5.1 Adapter 按能力拆分

当前现状：`AdapterFactory` 统一管理所有模型。

建议拆分为：
```
LLMAdapter      — 文本生成（DeepSeek、Kimi、OpenAI）
ImageAdapter    — 图像生成（FLUX、可灵图、即梦）
VideoAdapter    — 视频生成（可灵视频、Seedance、Veo）
TTSAdapter      — 语音合成（ElevenLabs、MiniMax）
MusicAdapter    — 音乐生成（待接入）
SoundAdapter    — 音效生成（待接入）
```

每个 Adapter 实现统一接口，支持：
- `validateKey()` — 验证 API Key
- `generate()` — 生成内容
- `getStatus()` — 查询任务状态
- `estimateCost()` — 估算成本

---

### 5.2 任务队列增强

当前：BullMQ 基础队列。

建议增强：
- **任务依赖链**：分镜生成 → 图像生成 → 视频生成 → 合成，自动串联
- **断点恢复**：任务失败后从失败步骤重试，不从头开始
- **并行度控制**：根据 API 限速自动调节并发数
- **WebSocket 推送**：实时进度推送到前端

---

### 5.3 资产管理升级

当前：项目内简单 CRUD。

建议升级为：
- **项目级资产库**：角色、场景、提示词、音乐、音效统一管理
- **跨项目复用**：角色/场景可在不同项目间复用
- **版本管理**：每次生成的资产自动保存版本，支持回溯
- **资产搜索**：按标签/类型/项目搜索资产

## 八、总结

### 我们的核心优势（保持）
- ✅ 模型中立 + 用户自带 API Key — 不绑定单一供应商
- ✅ 极速模式 — 3 分钟出草稿
- ✅ 暗色影院感 UI — 视觉风格差异化

### 需要补齐的关键差距
- 🔴 关键帧驱动 — 直接 Text-to-Video 不够可控
- 🔴 角色一致性 — 从单图升级到资产系统
- 🟡 多 Agent 编排 — 提升复杂任务的处理质量
- 🟡 剧本质量控制 — AI 自检剧本
- 🟡 成本可观测 — 用户需要知道钱花在哪

### 核心竞争策略
> **不追求功能数量最多，而是追求「3 分钟出草稿」体验最好 + 角色一致性最强。**

wind-comic 虽然功能最全（v10），但其复杂度也最高。我们的差异化应该聚焦在：
1. **极速模式**做到极致 — 比所有竞品都快
2. **角色一致性**做到最好 — 四视图+DNA锁
3. **模型中立**做到最灵活 — 用户自由切换模型
